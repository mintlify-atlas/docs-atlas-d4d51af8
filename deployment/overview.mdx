## Build Process

Astro generates a static site that can be deployed to any static hosting platform. The build process is simple:

```bash
npm run build
```

This command compiles your Astro project and outputs static files to the `dist/` directory. These files include:

- HTML files for each page
- Optimized CSS and JavaScript
- Static assets (images, fonts, etc.)
- Any other public files

## Quick Deployment Instructions

### GitHub Pages

<Steps>
  <Step title="Configure astro.config.mjs">
    Set the `site` and `base` options:
    ```js
    export default defineConfig({
      site: 'https://username.github.io',
      base: '/repository-name',
    });
    ```
  </Step>
  
  <Step title="Build your site">
    ```bash
    npm run build
    ```
  </Step>
  
  <Step title="Deploy">
    Use GitHub Actions (see [GitHub Actions workflow](/deployment/github-actions)) or push the `dist/` folder to the `gh-pages` branch.
  </Step>
</Steps>

### Netlify

<Steps>
  <Step title="Connect your repository">
    Sign in to Netlify and click "Add new site" → "Import an existing project".
  </Step>
  
  <Step title="Configure build settings">
    - Build command: `npm run build`
    - Publish directory: `dist`
  </Step>
  
  <Step title="Deploy">
    Click "Deploy site". Netlify will automatically rebuild on every push to your main branch.
  </Step>
</Steps>

### Vercel

<Steps>
  <Step title="Import your project">
    Sign in to Vercel and click "Add New" → "Project".
  </Step>
  
  <Step title="Configure project">
    Vercel automatically detects Astro projects. Verify:
    - Build command: `npm run build`
    - Output directory: `dist`
  </Step>
  
  <Step title="Deploy">
    Click "Deploy". Vercel provides preview deployments for every push.
  </Step>
</Steps>

### Cloudflare Pages

<Steps>
  <Step title="Create a new project">
    In the Cloudflare dashboard, go to Pages and click "Create a project".
  </Step>
  
  <Step title="Connect your Git repository">
    Authorize Cloudflare to access your repository.
  </Step>
  
  <Step title="Set build configuration">
    - Build command: `npm run build`
    - Build output directory: `dist`
  </Step>
  
  <Step title="Deploy">
    Click "Save and Deploy". Your site will be live on Cloudflare's global network.
  </Step>
</Steps>

## Custom Hosting (cPanel/FTP)

You can also deploy to traditional web hosting via FTP. This portfolio uses GitHub Actions to automatically deploy to cPanel.

<Warning>
  When deploying to custom hosting, ensure your hosting provider:
  - Supports static file hosting
  - Has proper file permissions configured
  - Serves `index.html` files by default
</Warning>

See the [GitHub Actions workflow](/deployment/github-actions) for automated deployment to cPanel via FTP.
