The `Layout.astro` file is the main template that wraps all pages in the portfolio. It provides the HTML structure, external dependencies, global styles, and animation scripts.

## Location

```plaintext
src/layouts/Layout.astro
```

## Props Interface

The layout accepts a single prop for the page title:

```typescript
interface Props {
    title: string;
}

const { title } = Astro.props;
```

**Usage**:
```astro
<Layout title="Ronald Santana - Portfolio">
    <!-- Page content -->
</Layout>
```

### External Dependencies

#### Bootstrap 5.3.3
- **CSS**: Loaded in the `<head>` for styling
- **JS**: Loaded at the end of `<body>` for interactive components
- **Icons**: Bootstrap Icons font for UI elements

#### Google Fonts - Onest
Variable font with weights from 100-900, used as the primary typography.

<Info>
The layout uses `preconnect` for Google Fonts to improve loading performance by establishing early connections to the font server.
</Info>

- **`overflow-x-hidden`**: Prevents horizontal scrolling
- **`<main>`**: Centered container with max-width (650px) and padding
- **`<slot />`**: Astro slot for injecting page content

## Intersection Observer for Scroll Animations

The layout includes a client-side script that powers scroll-triggered animations:

```javascript
document.addEventListener('DOMContentLoaded', function() {
    const observerOptions = {
        root: null,
        rootMargin: '0px',
        threshold: 0.1
    };

    const observer = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
            if (entry.isIntersecting) {
                entry.target.classList.add('visible');
                observer.unobserve(entry.target);
            }
        });
    }, observerOptions);

    // Observe elements with class fade-in-up
    document.querySelectorAll('.fade-in-up').forEach(element => {
        observer.observe(element);
    });
});
```

**How it works**:
1. Waits for DOM to load
2. Creates an Intersection Observer with 10% threshold
3. Finds all elements with `fade-in-up` class
4. When element enters viewport, adds `visible` class
5. Unobserves element after animation triggers (one-time effect)

<Tip>
Add the `fade-in-up` class to any element to enable scroll-triggered fade-in animations.
</Tip>

## Global Styles

The layout includes extensive global styles using Astro's `is:global` directive:

### CSS Variables

```css
:root {
    --primary: #1E1E1E;
    --bg: #f1f1f1;
}
```

### Base Styles

```css
body {
    background-color: var(--bg);
    width: 100%;
    height: 100vh;
    font-family: "Onest", sans-serif;
}

main {
    max-width: 650px;
    height: 100%;
}

a {
    color: var(--primary) !important;
}
```

### Animation Keyframes

Two primary animations are defined:

<CodeGroup>
```css fadeInUp
@keyframes fadeInUp {
    from {
        opacity: 0;
        transform: translateY(30px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}
```

```css fadeIn
@keyframes fadeIn {
    from {
        opacity: 0;
    }
    to {
        opacity: 1;
    }
}
```
</CodeGroup>

### Animation Classes

**Initial Load Animations**:
```css
.animate-on-load {
    opacity: 0;
    animation: fadeInUp 0.8s ease-out forwards;
}

.animate-delay-1 { animation-delay: 0.1s; }
.animate-delay-2 { animation-delay: 0.2s; }
.animate-delay-3 { animation-delay: 0.3s; }
.animate-delay-4 { animation-delay: 0.4s; }
```

**Scroll Animations**:
```css
.fade-in-up {
    opacity: 0;
    transform: translateY(30px);
    transition: opacity 0.6s ease-out, transform 0.6s ease-out;
}

.fade-in-up.visible {
    opacity: 1;
    transform: translateY(0);
}
```

<Note>
The difference between `animate-on-load` and `fade-in-up`:
- `animate-on-load`: Fires immediately when page loads
- `fade-in-up`: Fires when element scrolls into viewport
</Note>
