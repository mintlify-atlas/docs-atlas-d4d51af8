## Overview

The RSantana Portfolio uses a combination of CSS custom properties, Bootstrap 5, and custom global styles to create a clean, minimalist design. All global styles are defined in `src/layouts/Layout.astro`.

## CSS Custom Properties

The theme uses CSS custom properties (CSS variables) for easy color customization:

```css
:root {
  --primary: #1E1E1E;
  --bg: #f1f1f1;
}
```

### Available Variables

| Variable | Default Value | Usage |
|----------|---------------|-------|
| `--primary` | `#1E1E1E` | Primary text color and link colors |
| `--bg` | `#f1f1f1` | Background color for the entire site |

### Customizing Colors

To change the color scheme, modify the CSS custom properties in `src/layouts/Layout.astro`:

```css
:root {
  --primary: #2C3E50;  /* Dark blue-gray */
  --bg: #ECF0F1;       /* Light gray-blue */
}
```

<Tip>
Use variables like `var(--primary)` throughout your custom CSS to maintain consistency with the theme.
</Tip>

## Bootstrap Integration

The portfolio uses Bootstrap 5.3.3 loaded via CDN:

```html
<!-- Bootstrap CSS -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" 
      rel="stylesheet" 
      integrity="sha384-QWTKZyjpPEjISv5WaRU9OFeRpok6YctnYmDr5pNlyT2bRjXh0JMhjY6hW+ALEwIH" 
      crossorigin="anonymous">

<!-- Bootstrap Icons -->
<link rel="stylesheet" 
      href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.3/font/bootstrap-icons.min.css">
```

Bootstrap provides:
- Responsive grid system
- Utility classes (spacing, display, flex)
- Icon library (Bootstrap Icons)
- Pre-built components

<Note>
Bootstrap styles are loaded first, then overridden by custom global styles where needed.
</Note>

## Typography

### Font Setup

The portfolio uses **Onest** from Google Fonts:

```html
<!-- Fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Onest:wght@100..900&display=swap" 
      rel="stylesheet">
```

Applied globally:

```css
body {
  font-family: "Onest", sans-serif;
}
```

### Changing the Font

To use a different Google Font:

1. Visit [Google Fonts](https://fonts.google.com/)
2. Select your font and copy the embed code
3. Replace the Onest links in `Layout.astro`
4. Update the `font-family` in the global styles

```css
body {
  font-family: "Your Font Name", sans-serif;
}
```

## Global Styles

The main layout and styling:

```css
body {
  background-color: var(--bg);
  width: 100%;
  height: 100vh;
  font-family: "Onest", sans-serif;
}

main {
  max-width: 650px;
  height: 100%;
}

a {
  color: var(--primary) !important;
}
```

### Key Style Features

- **Centered Layout**: Content is centered with `max-width: 650px`
- **Link Colors**: All links use the `--primary` color
- **Overflow Control**: Body has `overflow-x-hidden` to prevent horizontal scrolling
- **Responsive Padding**: Main container uses `px-4` for horizontal spacing

## Modifying Global Styles

All global styles are defined in the `<style is:global>` section of `Layout.astro`. To add custom styles:

1. Open `src/layouts/Layout.astro`
2. Find the `<style is:global>` section
3. Add your custom CSS:

```css
<style is:global>
  :root {
    --primary: #1E1E1E;
    --bg: #f1f1f1;
    --accent: #3498db;  /* Add new variable */
  }
  
  /* Your custom styles */
  .my-custom-class {
    color: var(--accent);
  }
</style>
```

<Tip>
Use `is:global` to make styles available across all components and pages.
</Tip>

## Component-Specific Styles

For component-specific styles, use scoped `<style>` tags in individual `.astro` files:

```astro
---
// Component.astro
---
<div class="component">
  Content
</div>

<style>
  /* These styles only apply to this component */
  .component {
    padding: 2rem;
  }
</style>
```

## Best Practices

1. **Use CSS Variables**: Leverage custom properties for consistent theming
2. **Bootstrap First**: Use Bootstrap utilities before writing custom CSS
3. **Mobile-First**: Design for mobile, then enhance for larger screens
4. **Performance**: Keep the CDN approach for Bootstrap unless bundling is required
5. **Maintainability**: Group related styles together in the global stylesheet
