## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Node.js** (version 18.14.1 or higher)
- **npm**, **yarn**, or **pnpm** package manager
- A code editor (VS Code recommended)
- Git for version control

<Note>
  You can check your Node.js version by running `node --version` in your terminal.
</Note>

## Installation

Follow these steps to get your portfolio running locally:

<Steps>
  <Step title="Clone the repository">
    First, clone the RSantana Portfolio repository to your local machine:

    ```bash
    git clone <repository-url>
    cd rsantana
    ```
  </Step>

  <Step title="Install dependencies">
    Install the required dependencies using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>

    This will install Astro v5.13.10 and all necessary dependencies defined in `package.json`.
  </Step>

  <Step title="Start the development server">
    Launch the Astro development server:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    You should see output similar to:

    ```plaintext
    🚀 astro v5.13.10 started in XXms

    ┃ Local    http://localhost:4321/
    ┃ Network  use --host to expose
    ```
  </Step>

  <Step title="View your portfolio">
    Open your browser and navigate to:

    ```
    http://localhost:4321
    ```

    You should see the portfolio homepage with:
    - Navigation bar
    - Header section with developer introduction
    - Projects showcase section
    - Footer component
  </Step>
</Steps>

## Available Scripts

The portfolio comes with several npm scripts defined in `package.json`:

| Command | Description |
|---------|-------------|
| `npm run dev` | Starts the development server with hot module replacement |
| `npm run build` | Builds the production-ready static site |
| `npm run preview` | Previews the production build locally |
| `npm run astro` | Runs Astro CLI commands |

## Building for Production

When you're ready to deploy your portfolio, create a production build:

<CodeGroup>
  ```bash npm
  npm run build
  ```

  ```bash yarn
  yarn build
  ```

  ```bash pnpm
  pnpm build
  ```
</CodeGroup>

This command:
1. Generates static HTML files for all pages
2. Optimizes images and assets
3. Creates individual project pages from content collections
4. Outputs everything to the `dist/` directory

<Note>
  The build process uses Astro's static site generation. All routes are pre-rendered at build time for optimal performance.
</Note>

## Preview Production Build

Before deploying, you can preview your production build locally:

<CodeGroup>
  ```bash npm
  npm run preview
  ```

  ```bash yarn
  yarn preview
  ```

  ```bash pnpm
  pnpm preview
  ```
</CodeGroup>

This serves the contents of the `dist/` directory, allowing you to test the production build before deployment.

## Project Structure

After installation, you'll see the following structure:

```plaintext
rsantana/
├── public/              # Static assets (images, favicon)
│   ├── Edudu.png
│   └── favicon.svg
├── src/
│   ├── components/      # Reusable components
│   ├── content/         # Content collections
│   │   ├── config.ts    # Type-safe schemas
│   │   └── projects/    # Project markdown files
│   ├── layouts/         # Page layouts
│   └── pages/           # Route pages
├── astro.config.mjs     # Astro configuration
├── package.json         # Dependencies and scripts
└── tsconfig.json        # TypeScript configuration
```

## Development Workflow

### Hot Module Replacement

The development server includes HMR (Hot Module Replacement):
- Save any `.astro` file to see changes instantly
- Edit components and see updates without refresh
- Modify content collections and pages regenerate automatically

### Adding Projects

To add a new project to your portfolio:

1. Create a new Markdown file in `src/content/projects/`:

```markdown
---
title: 'Your Project Name'
description: 'Brief project description'
pubDate: '2024-01-15'
heroImage: '/project-image.png'
tag: 'Web Developer'
url: 'https://your-project-url.com/'
---

Your project content in Markdown format...
```

2. Add the project image to the `public/` directory
3. The project will automatically appear on the homepage and have its own route

<Warning>
  Make sure your project frontmatter matches the schema defined in `src/content/config.ts`. Required fields are `title`, `description`, and `pubDate`.
</Warning>

## Troubleshooting

### Port Already in Use

If port 4321 is already in use, specify a different port:

```bash
npm run dev -- --port 3000
```

### Node Version Issues

Ensure you're using Node.js 18.14.1 or higher. Use `nvm` to manage Node versions:

```bash
nvm install 18
nvm use 18
```

### Module Not Found Errors

Delete `node_modules` and reinstall:

```bash
rm -rf node_modules package-lock.json
npm install
```

<Note>
  For more help, check the [Astro documentation](https://docs.astro.build) or open an issue in the repository.
</Note>
